Filosofia de estudo

Fundamento não é superficial

Tipos, arrays e funções parecem simples, mas concentram a maior parte dos bugs reais: coerção implícita, valores nulos, índices ausentes, escopo e tratamento de erro.

Escreva no CLI

Antes de HTTP, banco ou framework, rode scripts pequenos com php arquivo.php. Observe entrada, saída, warnings, exceções e tipos retornados.

Composer como organização

Composer entra para organizar autoload, scripts e dependências, não para esconder a linguagem. Primeiro entenda o PHP; depois automatize.

Como estudar PHP do básico ao avançado

Use ciclos curtos: leia um conceito, escreva 3 exemplos próprios, quebre o código de propósito, corrija o erro e registre a armadilha no Obsidian. Só avance quando conseguir explicar o comportamento sem decorar.

  • Semana comum: 6h linguagem, 5h exercícios CLI, 3h refatoração, 2h leitura de documentação, 2h revisão.
  • Critério de domínio: você sabe prever o tipo de retorno, o erro provável e o formato da saída antes de executar.
  • Regra de ouro: se o conceito não virou função pequena, teste manual e anotação de armadilha, ele ainda não foi consolidado.

Fundamentos essenciais da linguagem

O coração do guia agora é consolidar comportamento de runtime. Antes de discutir arquitetura, você deve dominar como o PHP recebe valores, converte tipos, avalia expressões, chama funções e reporta falhas.

  • Use declare(strict_types=1) em todo exemplo novo.
  • Prefira comparação estrita: === e !==.
  • Modele retornos: nunca deixe uma função “talvez retornar qualquer coisa”.
  • Trate null como decisão explícita, não como valor de escape.
<?php declare(strict_types=1);

function normalizeQuantity(mixed $value): int
{
    if (! is_numeric($value)) {
        throw new InvalidArgumentException('Quantidade deve ser numérica.');
    }

    $quantity = (int) $value;

    if ($quantity <= 0) {
        throw new DomainException('Quantidade deve ser positiva.');
    }

    return $quantity;
}

Composer e autoload

Composer aparece depois que a linguagem básica estiver em andamento. O objetivo é organizar código próprio, carregar classes e automatizar comandos, sem transformar dependências em muleta.

// composer.json mínimo para estudos
{
  "autoload": {
    "psr-4": { "App\\": "src/" }
  },
  "scripts": {
    "start": "php public/index.php",
    "autoload": "composer dump-autoload"
  }
}

match para regra explícita

$label = match ($status) {
    'paid' => 'Pago',
    'open' => 'Aberto',

Array para objeto quando há regra

final readonly class Email
{
    public function __construct(public string $value)
    {
        if (! filter_var($value, FILTER_VALIDATE_EMAIL)) {
            throw new InvalidArgumentException('E-mail inválido.');
        }
    }
}
